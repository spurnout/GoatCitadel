# Review Checklist

Use this checklist before approving any proposal.

## Core Questions

- Is the pattern explicit rather than guessed?
- Has it recurred at least 3 times?
- Has it appeared across 2 or more tasks or sessions?
- Is the target surface correct?
- Is the proposal concise?
- Would it reduce future friction or failure?
- Is it free of sensitive data?
- Does it avoid increasing permissions?
- Does it avoid direct code or config mutation?

## Proposal Quality

A good proposal should have:
- a short summary
- a clear target
- evidence references
- suggested text or outline
- risk level
- rollback note if relevant

## When To Reject

Reject or leave pending if:
- it is only situational
- it encodes a temporary preference
- it tries to smuggle in broader access
- it duplicates existing guidance
- it belongs in a log, not a durable target
- it reflects a single emotional moment instead of a stable pattern
